# 업무일지

`worklog`는 실제 요청과 수행 결과를 보관한다.

- `requests-log.md`: 요청과 처리 결과의 시간순 기록
- `YYYY-MM-DD.md`: 날짜별 업무 목적, 수행 내용, 결과와 후속 작업

앞으로 할 일과 구현 지시는 `docs/backlogs/`에서 관리한다.

## 날짜별 업무일지 형식

```md
# YYYY-MM-DD 업무일지

- 작성자:
- 작업일: YYYY-MM-DD

## 작업 목적

-

## 수행 내용

-

## 실행 결과

-

## 산출물

-

## 후속 작업

-
```
