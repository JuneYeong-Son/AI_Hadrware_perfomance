## 작업 목적

- NVIDIA 디자인 시스템(`docs/DESIGN-nvidia.md`)을 데스크톱 앱 UI에 적용한다.
- 사용자별 계정과 측정 결과 진위 증명을 위한 백엔드 서버를 구축한다.
- 사용자들의 벤치마크/TFLOPS를 서버에 기록해 성능 데이터를 분석할 수 있게 한다.
- 데스크톱 앱에 로그인/로그아웃과 결과 업로드(공유 검증 코드)를 연동한다.

## 수행 내용

- 데스크톱 테마를 NVIDIA 토큰으로 교체했다: 그린(#76b900) 단일 강조, 2px 각진
  모서리, 헤어라인(#cccccc) 테두리, 그림자 제거, 카드 좌상단 그린 코너 스퀘어.
  (`theme.py`, `widgets.py`, `buyer/main_window.py`의 인라인 상태색 정렬)
- FastAPI 백엔드를 신설했다(`application/backend/`): SQLAlchemy + SQLite + JWT.
- 회원가입/로그인/로그아웃을 구현했다. 로그아웃은 `token_version` 증가로 기존
  토큰을 서버측에서 즉시 무효화한다(bcrypt 비밀번호 해시).
- 측정 진위 증명을 구현했다: 기기 등록 시 공유용 코드(`NV-XXXX-XXXX`)와 기기 전용
  HMAC 시크릿을 발급하고, 측정 제출 시 핵심값을 해시·서명해 공유용 검증 코드
  (`M-XXXX-XXXX`)를 만든다. 공개 `/api/verify` 엔드포인트로 위조 여부를 확인한다.
- 성능 분석 API를 구현했다: 모델·프로토콜별 TFLOPS 분포(avg/median/p10/p90/max),
  리더보드, 내 백분위. 엔진 규칙대로 같은 `protocol_id` + `valid` 결과끼리만 집계.
- 데스크톱에 로그인 게이트와 상태바 계정바(이름 + 로그아웃/로그인)를 추가했다.
  메인 윈도우 클래스는 수정하지 않고 `app._run`에서 주입한다.
- 로컬 토큰 저장으로 자동 로그인을 구현했다(로그아웃 시 삭제, 오프라인 모드 유지).
- GPU Check 결과 화면에 "결과 기록 · 공유(진위 증명)" 카드와 업로드 버튼을 추가했다.
  GPU UUID 원본은 전송하지 않고 해시만 보내며, valid 측정만 기록한다.

## 실행 결과

- 백엔드 스모크 테스트: 회원가입·로그인·me·중복가입(409)·오답(401)·기기등록(멱등)·
  측정 제출/서명·공개 검증·위조코드(404)·분석(모델/리더보드/백분위)·로그아웃
  토큰 무효화 전부 통과.
- 데스크톱 ApiClient(실서버 대상): signup/login/me/오답거부/로그아웃무효화/오프라인
  에러 통과.
- 오프스크린 Qt 테스트: 로그인 다이얼로그·계정바·토큰 저장 왕복 통과.
- 오프스크린 업로드 통합(실서버): 실제 `GpuCheckWindow`에서 업로드 → UI에 검증
  코드 표시 → 백엔드 `/verify`가 동일 코드로 8.22 TFLOPS 확인 → 클립보드 복사 통과.
